# 安全政策 (Security Policy)

## 支持版本

我们积极维护以下版本的安全更新：

| 版本 | 支持状态 |
|-------|----------|
| 1.0.x | ✅ 支持中 |
| < 1.0 | ❌ 不支持 |

## 报告漏洞

我们非常重视安全问题。如果您发现 kaolalabot 中的安全漏洞，请按照以下步骤报告：

### 📧 联系方式

- **报告邮箱**: security@kaolalabot.dev（待设置）
- **GitHub Security Advisories**: 通过 GitHub 的 Private Vulnerability Reporting 功能报告

### 📝 报告格式

为了帮助我们更好地理解并修复问题，请在报告中包含以下信息：

- 漏洞类型和描述
- 受影响的版本
- 复现步骤
- 潜在影响
- 可能的修复建议（如有）

### 🔒 负责任的披露

我们采用负责任的披露政策：

1. **初步响应**：在收到报告后 48 小时内确认
2. **评估**：在 7 天内评估漏洞的严重性
3. **修复开发**：根据严重程度制定修复计划
4. **公开发布**：在修复发布后公开披露

### 🛡️ 安全最佳实践

使用 kaolalabot 时，请遵循以下安全建议：

#### API 密钥管理
- 永远不要将 API 密钥提交到版本控制
- 使用环境变量或本地配置文件（已添加到 .gitignore）
- 定期轮换密钥
- 使用最小权限原则

#### 网络安全
- 在生产环境中使用 HTTPS
- 限制回调端口的访问
- 使用防火墙规则保护服务

#### 数据安全
- 对敏感数据进行加密
- 定期备份重要数据
- 审查第三方依赖的安全性

### 📢 安全公告

重大安全问题修复后，我们将通过以下方式发布安全公告：

- GitHub Security Advisories
- Release Notes
- 项目 README 更新

我们感谢所有帮助提高 kaolalabot 安全性的研究人员和用户。
